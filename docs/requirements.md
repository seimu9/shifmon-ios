# シフモン 要件定義書

## 1. プロジェクト概要

### 1.1 アプリ名

シフモン / Shifmon

### 1.2 コンセプト

シフモンは、アルバイトのシフト管理・給料計算・カレンダー連携に、キャラクター育成要素を組み合わせたiPhone向けアプリである。

コンセプトは「働く予定を、ちょっと楽しみに変える」。

### 1.3 開発目的

既存のシフト管理アプリでは、シフト登録・給料計算はできるが、以下の課題がある。

- もらったシフトを手入力するのが面倒
- 標準カレンダーへの転記が手間
- 給料見込みは見えるが、働くモチベーションにつながりにくい
- 継続して使いたくなる楽しさが少ない

シフモンでは、シフト管理を単なる記録作業ではなく、育成・可視化・達成感を通じて前向きな体験に変えることを目指す。

## 9. テスト観点

### 9.1 シフト追加

- 勤務先を入力して保存できる
- 勤務先未入力でも保存できる
- 終了時間が開始時間より前の場合は保存できない
- 時給が未入力または0以下の場合は保存できない
- 休憩時間が未入力または不正な場合は保存できない
- 保存後にホーム画面へ戻る

### 9.2 給料計算

- 勤務時間から休憩時間を引いて計算される
- 時給と実働時間から見込み給料が表示される
- 今月分のみ月間合計に反映される

### 9.3 保存

- アプリを再起動しても登録済みシフトが残る
- 複数件のシフトを保存できる

## 12. スクリーンショット読み取り・AI解析方針

### 12.1 基本方針

シフモンでは、ジョブカン・freee・その他シフト管理サービスのスクリーンショットからシフト情報を読み取り、アプリ内のシフト登録とiPhone標準カレンダー登録を自動化することを目指す。

ただし、最初から完全自動登録は行わず、必ずユーザー確認画面を挟む。

### 12.2 段階的な実装方針

#### v0.1：端末内OCR

Apple Visionを利用し、iPhone端末内で画像から文字を読み取る。

目的は以下とする。

- スクリーンショットを選択できる
- 画像内の文字を読み取れる
- 読み取り結果を画面に表示できる

この段階では、まだシフト候補への自動変換は行わない。

#### v0.2：OCR結果からシフト候補を抽出

Apple Visionで読み取ったテキストから、日付・開始時刻・終了時刻の候補を抽出する。

まずはジョブカン形式を対象とする。

抽出対象の例：

```json
[
  {
    "date": "2026-07-03",
    "startTime": "12:10",
    "endTime": "23:00",
    "workplaceName": "未設定"
  }
]
```

#### v0.3：確認画面と一括登録

抽出したシフト候補を確認画面に表示する。

ユーザーは以下を行える。

- 候補を確認する
- 誤認識を修正する
- 不要な候補を削除する
- 問題なければ一括登録する

登録時は以下へ反映する。

- Shifmon内のSwiftData
- iPhone標準カレンダー

#### v1.0：AI APIによる汎用解析

将来的には、AI APIを利用してジョブカン以外の形式にも対応する。

基本方針は以下とする。

1. まずApple Visionで端末内OCRを行う
2. OCR結果のテキストだけをAI APIへ送る
3. AI APIがシフト候補JSONを返す
4. ユーザーが確認・修正する
5. 問題なければ一括登録する

この方式により、画像そのものを外部に送る頻度を減らし、料金・プライバシー・通信量を抑える。

#### v1.1：高精度AI画像解析

OCR結果だけでは解析が難しい場合に限り、画像そのものをAI APIに送信する高精度モードを用意する。

高精度モードは以下の扱いとする。

- ユーザーが明示的に実行した場合のみ使用する
- 画像をAI解析に送信することを事前に説明する
- 解析後は必ず確認画面を表示する
- 自動登録は行わない
- 将来的には広告視聴や回数制限と組み合わせる

### 12.3 推奨API方針

第一候補は、テキスト解析と画像解析の両方に対応できるAI APIとする。

初期の推奨構成は以下。

- 端末内OCR：Apple Vision
- OCRテキストの構造化：AI API
- 画像解析の高精度モード：AI API
- APIキー管理：iOSアプリ内には直接保存せず、バックエンド経由で利用する

### 12.4 安全性・プライバシー方針

シフト画像には、勤務先・勤務日・勤務時間・給与情報などの個人情報が含まれる可能性がある。

そのため、以下を守る。

- AI解析はユーザー操作時のみ実行する
- 画像送信前に説明を表示する
- 通常モードでは画像ではなくOCRテキストのみ送信する
- 画像を送る場合は高精度モードとして明示する
- APIキーをiOSアプリに直接埋め込まない
- 解析結果は必ずユーザーが確認してから登録する
- 将来的にApp Store公開する場合はプライバシーポリシーを用意する

### 12.5 料金方針

AI API利用には従量課金が発生する可能性があるため、以下の設計で料金を抑える。

- 通常はApple Visionによる端末内OCRを使う
- AI APIには画像ではなくOCRテキストを送る
- 画像解析は失敗時や高精度モードに限定する
- 将来的には月間無料回数を設定する
- 高精度AI解析は広告視聴や回数制限と組み合わせる
